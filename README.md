# RiverBorn

RiverBorn 是一个用 Rust + Bevy 制作的“德扑式地牢牌桌”游戏。

它的核心想法是：把传统牌局里的抽象牌力，变成一支具体队伍的地牢生存能力。玩家根据自己的隐藏冒险者、公共牌、地牢威胁和下注局势判断要不要继续投入。你可以用强阵容价值下注，也可以诈唬逼退别人；但即使其他人都弃牌，最后留下的人也必须真的跑过地牢，才能拿走底池。

## 游戏概念

每一手牌由三层组成：

- **2 张隐藏冒险者**：每个玩家自己的底牌，只有本人可见。
- **5 张公共牌**：随下注阶段逐步翻开，可能是角色、绑定装备或战术消耗品。
- **3 个地牢节点**：随公共牌逐步翻开，依次是小怪、精英/环境、Boss。

摊牌时，玩家组成一支 5 张牌的队伍：

```text
2 张底牌冒险者 + 从公共牌中选择 3 张
```

AI 会自动选择最优的 3 张公共牌。人类玩家如果坚持到摊牌，需要手动点击 3 张公共牌，然后确认组队。

## 当前实现流程

当前版本实现的是单机现金局式牌桌：

1. 进入主菜单。
2. 选择玩家人数，支持 3-6 人。
3. 创建 1 名人类玩家 + 若干 AI 的牌桌。
4. 每个座位初始买入一锅：`1000` 筹码。
5. 每手随机一个场景，发隐藏冒险者，收前注，然后进入下注和翻牌流程。
6. 到摊牌时，未弃牌玩家组队跑地牢。
7. 有玩家通关时，通关者中优势最高的人拿走底池。
8. 无人通关时，底池滚入下一手。
9. 点击“下一手”继续整场对局。

`Match` 层会跨手保留筹码、负债、手数、前注增长、场景状态和滚存彩池。

## 单手牌阶段

```text
PreFlop  - 玩家只看到自己的 2 张隐藏冒险者
Flop     - 翻 3 张公共牌 + 1 个小怪节点
Turn     - 翻 1 张公共牌 + 1 个精英/环境节点
River    - 翻 1 张公共牌 + Boss
Showdown - 选择公共牌、组队、跑地牢、结算
Done     - 本手已结算，等待下一手
```

下注动作：

- `Check`
- `Call`
- `Raise`
- `All-in`
- `Fold`

如果其他玩家全部弃牌，只剩一个人，本手不会直接结束。系统会补齐完整牌面和完整地牢线，让最后留下的人进入摊牌跑本。诈唬成功只是让别人撤退，不代表可以不打 Boss。

## 战斗规则

战斗逻辑位于 `game_core`，不依赖 Bevy。

组队规则：

- 队伍固定包含 2 张底牌冒险者。
- 从已翻开的公共牌中选择 3 张。
- 角色牌提供自身 `P` 和 `H`。
- 装备牌只有在队伍中存在绑定职业时才生效。
- 消耗品提供特殊战术效果。
- 同职业对子/三条提供额外加成。
- 场景会影响队伍或地牢。

跑地牢时逐节点结算：

- 队伍 `power` 需要压过每个节点的有效威胁。
- 队伍 `health` 需要扛住每个节点的有效威胁。
- 扛过节点后，`health` 扣除该节点威胁。
- 通关玩家按战后优势值比较。

当前主要加成和效果：

- 同职业对子：`+6P +6H`
- 同职业三条：`+12P +12H`
- Fireball：降低小怪节点威胁。
- Smoke Bomb：降低第一个地牢节点威胁。
- Cleric / Healing Potion / Purify：提供净化，免疫毒沼叠毒。
- All-in：让 Boss 狂暴。
- 惊动值：提高精英、环境、Boss 的威胁。

## 场景

每手随机一个场景：

- Ice Field：固定降低队伍战力。
- Stone Arena：固定提高队伍战力。
- Marsh：没有净化时逐节点叠毒。
- Tomb：提高战力但降低生命。
- Volcano：每推进一个下注轮都会增加惊动值。
- Wheat Field：Ranger 和 Mage 获得额外战力。

Bevy 表现层会根据当前场景切换牌桌背景。

## 操作

牌局中：

```text
Q    Check
W    Call
E    Raise +20
R    Fold
Esc  暂停菜单
Tab  记账面板
```

摊牌时，如果人类玩家仍在手牌中，需要点击 3 张公共牌，再点击 `确认组队` 完成本手结算。

## 当前 UI 功能

- Bevy 0.15 牌桌界面。
- 主菜单、设置占位页、多人模式占位页。
- 单机人数选择，支持 3-6 人。
- 场景背景切换。
- 卡牌图、卡背、头像、数值标签。
- 菜单/游戏 BGM 和翻牌音效。
- 暂停菜单。
- 摊牌结算面板，展示地牢威胁、队伍 P/H、通关结果和优势值。
- 记账面板，展示筹码、负债、净身价和水上/水下。
- 人类筹码不足时显示“借一锅”按钮。

## 代码结构

```text
crates/
  game_core/
    纯规则库：卡牌、下注、战斗、现金局经济、AI、确定性 RNG。

  riverborn/
    Bevy 客户端：菜单、牌桌 UI、输入、音频、资源加载、结算面板。

  riverborn_net/
    未来联机用的 serde/JSON 协议类型。
```

`game_core` 刻意不依赖 Bevy，这样规则可以快速测试，也方便未来被权威服务器复用。

## 联机状态

联机功能尚未完整实现。

当前 `riverborn_net` 已经定义了大厅阶段的协议消息：

- hello / welcome
- 创建房间
- 拉取房间列表
- 加入房间
- 离开房间
- 房主开始游戏
- 房间状态更新

预期联机架构：

```text
Bevy 客户端 -> 发送玩家操作和 UI 输入
服务器      -> 持有 Match/Game，验证动作，裁剪隐藏信息
game_core   -> 复用确定性规则
```

服务器不能直接把完整 `Game` 广播给所有客户端，因为里面包含所有玩家的隐藏底牌。正确做法是给每个客户端发送它自己可见的牌桌视图。

## 运行

单机客户端：

```bash
cargo run -p riverborn
```

核心规则测试：

```bash
cargo test -p game_core
```

联机协议测试：

```bash
cargo test -p riverborn_net
```

注意：当前 workspace manifest 中包含计划中的 `riverborn_server` member。如果本地 checkout 里还没有这个 crate，`cargo` 会报 workspace member 缺失。需要补上服务端 crate，或者临时从 workspace members 里移除它。

## 资源

运行时资源位于：

```text
crates/riverborn/assets/
```

主要资源目录：

- `cards/community/`：冒险者、装备、消耗品。
- `cards/dungeon/`：地牢节点和 Boss。
- `cards/card_back*.png`：随机卡背。
- `table/`：场景背景。
- `avatars/`：玩家和 AI 头像。
- `audio/`：菜单音乐、游戏音乐和音效。

Bevy 客户端通过 `AssetServer` 按相对路径加载这些资源。

## 当前定位

RiverBorn 当前更接近一个可玩的原型：

- 下注循环已经能跑通。
- 地牢生存结算已经能跑通。
- AI 可以自动完成牌局。
- 多手现金局、负债、利息、滚存彩池已经实现。
- UI 可用，但动画和表现还比较轻。

可能的下一步：

- 重新设计牌池数量和数值。
- 给卡牌增加明确名称，避免只靠 art id 和数值区分。
- 重新整理地牢 `health/threat` 的含义和结算公式。
- 增加发牌、翻牌、弃牌、All-in、结算等动画事件。
- 完成权威服务器和真正的联机模式。
